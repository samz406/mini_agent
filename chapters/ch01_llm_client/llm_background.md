# 背景介绍：大语言模型与人工智能的演进

> 📖 这是第一章的前置背景阅读，帮助你在动手封装 LLM 客户端之前，先建立起对"大语言模型是什么、从哪里来"的整体认知。

---

## 一、人工智能的发展简史

### 1.1 符号主义时代（1950s–1980s）：让机器"照规则走"

人工智能的概念诞生于 1956 年的达特茅斯会议，彼时研究者们相信：**只要把人类的知识和推理规则编写成程序，机器就能变得智能**。

这一流派被称为**符号主义（Symbolic AI）**或**专家系统（Expert System）**。它的核心思想是：把专家知识转化成 `IF-THEN` 规则，让计算机用逻辑推理得出结论。

**典型应用：**
- MYCIN（1970s）：诊断细菌感染的医疗专家系统，其规则库由医生手动输入
- Prolog：专门为逻辑推理设计的编程语言
- IBM 深蓝（Deep Blue）：通过穷举搜索和手工编写的棋局评估函数，于 1997 年击败国际象棋世界冠军卡斯帕罗夫

**局限：** 规则需要人工编写，维护成本极高。面对真实世界的模糊性与复杂性（比如自然语言），规则库很快就变得无法管理。到了 1980s 末，AI 进入第一次"寒冬"。

---

### 1.2 统计学习时代（1980s–2010s）：让机器"从数据中学"

研究者们开始转变思路：**与其手写规则，不如让机器自己从大量数据中总结规律**。

这催生了**机器学习（Machine Learning）**这一领域。几个关键里程碑：

| 时间 | 里程碑 | 意义 |
|------|--------|------|
| 1986 | 反向传播算法（Backpropagation）重新发现 | 神经网络可以通过梯度下降自动调整权重 |
| 1990s | 支持向量机（SVM）、决策树 | 统计机器学习方法在分类问题上表现优异 |
| 1998 | LeNet（Yann LeCun） | 卷积神经网络成功识别手写数字 |
| 2001 | 随机森林（Random Forest） | 集成学习方法在表格数据上大放异彩 |
| 2006 | Hinton 提出深度信念网络（DBN） | 打破了"深层网络难以训练"的偏见 |

这一时期，自然语言处理（NLP）的主流方法是**统计语言模型**——用 N-gram 统计相邻词的共现概率，预测下一个词是什么。虽然有效，但受限于固定长度的上下文窗口，无法捕捉长距离依赖关系。

---

### 1.3 深度学习革命（2012–2017）：神经网络的复兴

2012 年是人工智能历史上的转折点。

**AlexNet 时刻（2012）：** Alex Krizhevsky 等人在 ImageNet 图像识别竞赛中，用 GPU 训练的深度卷积神经网络将错误率从 26% 直接降至 15.3%，将第二名远远甩在身后。这一壮举让整个行业意识到：**深度神经网络 + 大数据 + GPU 算力，就是打开智能的钥匙**。

随后几年，深度学习席卷各个领域：

- **计算机视觉**：ResNet（2015）、VGG、GoogLeNet，机器识图能力超越人类
- **语音识别**：端到端深度学习取代了 HMM+GMM 的传统管线，语音识别错误率大幅下降
- **自然语言处理（NLP）**：Word2Vec（2013）、GloVe 等词向量方法，让机器第一次能够在高维空间中"理解"词的含义

**Word2Vec 的直觉：**
```
"国王" - "男人" + "女人" ≈ "女王"
```
词向量把单词映射为数百维的浮点数向量，语义相似的词在空间中距离相近。这是机器理解语言的重要一步。

---

### 1.4 序列到序列与注意力机制（2014–2017）：理解"上下文"

语言是有序列结构的——理解一句话，需要记住前面说了什么。

**循环神经网络（RNN）与 LSTM（2014 前后）** 被设计来处理序列数据。Google 用 LSTM 构建的神经机器翻译（NMT）系统显著提升了翻译质量。

然而 RNN 有一个致命弱点：**梯度消失**——随着序列变长，早期信息在反向传播过程中逐渐消失，模型无法有效"记住"很长的上下文。

**注意力机制（Attention Mechanism，2015）** 的出现解决了这一问题。它允许模型在处理序列中每个位置时，**直接"注意到"序列中任何其他位置**，而不必通过逐步传递。

```
传统 RNN：A → B → C → D（信息需要一步步传递，远距离信息容易丢失）
注意力机制：D 可以直接"看"A、B、C（全局依赖，一步到位）
```

这为后来的 Transformer 奠定了基础。

## 二、Transformer：大语言模型的基石

### 2.1 "Attention Is All You Need"（2017）

2017 年，Google 的研究团队发表了一篇名为 **"Attention Is All You Need"** 的论文，提出了 **Transformer** 架构。

这篇论文的核心洞见是：**把 RNN 完全丢掉，只用注意力机制（Self-Attention）来建模序列中的依赖关系**。

**Transformer 的核心组件：**

| 组件 | 作用 |
|------|------|
| 多头自注意力（Multi-Head Self-Attention） | 让每个词同时关注序列中所有其他词，捕捉全局依赖 |
| 前馈网络（Feed-Forward Network） | 对每个位置的表示进行非线性变换 |
| 位置编码（Positional Encoding） | 为纯注意力机制补充位置信息（注意力本身是无序的） |
| 层归一化（Layer Normalization） | 稳定训练过程 |
| 编码器-解码器结构（Encoder-Decoder） | 编码器理解输入，解码器生成输出 |

**为什么 Transformer 更好？**

1. **并行化**：注意力计算可以完全并行，GPU 能充分发挥，训练速度远超 RNN
2. **长距离依赖**：任意两个位置的信息流路径长度均为 1，不会遗忘
3. **可扩展性**：增加层数、增大模型，性能持续提升（"扩展定律"）

---

### 2.2 BERT 与 GPT：两种预训练范式（2018）

Transformer 出现后，研究者们发现：**在海量文本上预训练 Transformer，然后在特定任务上微调，效果远好于从头训练**。这开创了"**预训练 + 微调**"的时代。

两种主要范式同年诞生：

#### BERT（2018，Google）—— 双向编码器

BERT（Bidirectional Encoder Representations from Transformers）用 Transformer 的**编码器部分**，通过以下两个任务在大量文本上预训练：

- **掩码语言模型（MLM）**：随机遮住 15% 的词，让模型根据上下文预测被遮住的词
  ```
  输入：The [MASK] sat on the mat.
  目标：cat
  ```
- **下一句预测（NSP）**：判断两句话是否在原文中相邻

BERT 的特点是**双向**——同时看左侧和右侧的上下文，非常适合理解类任务（分类、问答、命名实体识别）。

#### GPT（2018，OpenAI）—— 自回归解码器

GPT（Generative Pre-trained Transformer）用 Transformer 的**解码器部分**，通过**预测下一个词**进行预训练：

```
输入：The cat sat on the
预测：mat
```

GPT 的特点是**单向**（从左到右），自然适合文本生成任务。

**关键洞察：** 这两种范式孰优孰劣，在当时并无定论。但 GPT 的路线最终演变出了大语言模型的主流方向。

## 三、大语言模型的崛起

### 3.1 规模定律（Scaling Laws，2020）

OpenAI 在 2020 年发表了一篇关键论文《Scaling Laws for Neural Language Models》，揭示了一个惊人的规律：

> **语言模型的性能，可以用模型参数量、数据量、计算量来精确预测，且这三者均与性能呈幂律关系。**

简单说：**模型越大、数据越多、算力越强，模型就越聪明，且没有明显的"天花板"**。

这一发现彻底改变了研究方向——既然规模能带来能力，那就堆规模。

### 3.2 GPT 系列的演进

| 版本 | 时间 | 参数量 | 里程碑意义 |
|------|------|--------|-----------|
| GPT-1 | 2018 | 1.17 亿 | 证明预训练 + 微调范式有效 |
| GPT-2 | 2019 | 15 亿 | 生成能力强到 OpenAI 一度不敢公开发布 |
| GPT-3 | 2020 | 1750 亿 | 涌现出少样本学习（Few-Shot）能力，震惊学界 |
| InstructGPT | 2022 | 1750 亿 | 引入 RLHF，让模型"听话"而非只生成流畅文本 |
| ChatGPT | 2022.11 | 未公开 | 5 天 100 万用户，重新定义人机交互 |
| GPT-4 | 2023 | 未公开（估计万亿级） | 多模态、通过律师资格考试等专业水准 |

### 3.3 "涌现能力"：量变引发质变

当模型规模超过某个阈值时，会突然具备之前从未有过的能力——这被称为**涌现能力（Emergent Abilities）**。

典型的涌现能力包括：

- **少样本学习（Few-Shot Learning）**：GPT-3 只需在 prompt 中给几个例子，就能完成从未专门训练过的任务
- **思维链推理（Chain-of-Thought）**：让模型"一步步思考"，数学推理准确率大幅提升
- **指令跟随（Instruction Following）**：理解并执行复杂的自然语言指令
- **代码生成**：从自然语言描述自动生成可运行的程序

这些能力的出现并非"明确设计"的结果，而是**规模带来的自发涌现**。这让研究者和工程师们意识到：大语言模型不只是"更好的自动补全"，它是一种全新的通用计算基础设施。

### 3.4 RLHF：让模型对齐人类意图

规模大并不够——GPT-3 虽然强大，却经常说有害内容、编造事实、或给出有偏见的回复。

**基于人类反馈的强化学习（RLHF，Reinforcement Learning from Human Feedback）** 是让模型"对齐"人类价值观的关键技术：

```
1. 监督微调（SFT）：用人工标注的高质量对话示例微调模型
        ↓
2. 奖励建模（RM）：训练一个奖励模型，学习"什么是好回复"
        ↓
3. PPO 强化学习：用奖励模型作为反馈信号，进一步优化语言模型
```

InstructGPT 和 ChatGPT 的核心突破正是 RLHF——模型不只是生成流畅的文字，而是**生成有帮助的、无害的、诚实的回复**（HHH：Helpful, Harmless, Honest）。

ChatGPT 的发布掀起了一场席卷全球的 AI 热潮，大量大语言模型相继涌现：

| 模型 | 机构 | 特点 |
|------|------|------|

| Gemini 系列 | Google | 原生多模态，与 Google 生态深度整合 |

同时，**API 经济**快速形成：OpenAI、Anthropic、Google 等公司以 API 的形式对外开放模型能力，开发者只需几行代码即可接入世界顶级的 AI 能力，无需自己训练模型——这正是我们这门课程的起点。

## 五、大语言模型的本质与局限

### 5.1 LLM 在做什么？

从技术本质上看，大语言模型是一个**条件概率分布**：

```
P(下一个 token | 所有之前的 token)
```

给定一段文本，模型计算所有可能的下一个词（token）的概率，然后采样输出。生成文本的过程就是不断重复这个过程：预测 → 输出 → 把输出加入上下文 → 再预测。

尽管这看起来很简单，但在海量数据训练下，模型内化了大量关于世界的知识、语言规律、推理模式，因此能完成极其复杂的任务。

### 5.2 核心局限

理解 LLM 的局限，是构建可靠 Agent 系统的前提：

| 局限 | 说明 | Agent 如何应对 |
|------|------|--------------|
| **知识截止（Knowledge Cutoff）** | 训练数据有截止日期，不知道最新信息 | 工具调用：让 Agent 实时搜索 |
| **幻觉（Hallucination）** | 自信地编造不存在的事实 | RAG（检索增强生成）+ 验证机制 |
| **上下文窗口有限** | 无法"记住"超过一定长度的对话 | 记忆系统（本课程第四、五章） |
| **无法执行代码** | 只能生成代码，不能运行并观察结果 | 工具调用（本课程第三章） |
| **单轮思维** | 一次性生成完整回复，无法迭代修正 | Agent 循环（本课程第二章） |

**这正是为什么我们需要构建 Agent**——将 LLM 的语言理解与推理能力，和外部工具、记忆系统、循环执行机制结合，弥补其先天局限。
